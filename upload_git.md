
بصفتك خبير DevOps ومطور برمجيات محترف، أريدك أن تقوم برفع مشروعي الحالي إلى مستودع (Repository) على GitHub، مع الالتزام الصارم بالقواعد التالية:

1. تحليل المشروع وتنسيقه:
   - قم بفحص هيكل المجلدات الحالي. إذا لم يكن متوافقاً مع معايير GitHub (مثل نقص ملف README.md أو عدم وجود بنية منظمة)، قم باقتراح التعديلات أو تنفيذها مباشرة لتحسين مظهر المشروع.
   - تأكد من وجود ملف README.md احترافي يشرح المشروع وكيفية تشغيله.

2. الأمان والخصوصية (هام جداً):
   - يمنع منعاً باتاً رفع أي ملفات تحتوي على معلومات حساسة مثل (API Keys, Secrets, Passwords).
   - ابحث عن ملفات .env أو أي ملفات إعدادات محلية وقم بإضافتها فوراً إلى ملف .gitignore.
   - إذا لم يكن هناك ملف .gitignore، قم بإنشاء واحد مخصص لنوع لغة البرمجة المستخدمة في هذا المشروع (مثلاً: Node.js, Python, Flutter... إلخ).
   - تأكد من عدم رفع مجلدات المكتبات الضخمة مثل (node_modules, venv, target).

3. اختيار الملفات الأساسية فقط:
   - ارفع فقط الملفات الضرورية لعمل الكود المصدري. لا ترفع ملفات النظام (.DS_Store) أو ملفات الـ Cache أو ملفات الـ Build المؤقتة.

4. آلية التنفيذ والتواصل:
   - ابدأ بفحص ما إذا كان Git مفعلاً في المجلد. إذا لم يكن، فقم بتهيئته (git init).
   - إذا كنت تحتاج إلى "GitHub Personal Access Token" أو "Remote URL" أو أي صلاحية لا تملكها، توقف واطلبها مني بوضوح.
   - قبل عملية الـ Push النهائية، أعطني ملخصاً بالملفات التي سيتم رفعها والملفات التي تم استثناؤها للأمان.

5. خطوات العمل:
   - إنشاء/تعديل .gitignore.
   - تنظيم الهيكل (إذا لزم الأمر).
   - إضافة الملفات (git add .).
   - عمل أول Commit برسالة وصفية احترافية.
   - ربط المستودع ورفع الملفات (git push).

ابدأ الآن بفحص المشروع وأخبرني بالخطوة الأولى.


نصائح إضافية لضمان نجاح العملية:

1. ملف الـ .env: إذا كان لديك ملف .env يحتوي على مفاتيح حقيقية، يفضل دائماً أن تصنع ملفاً آخر اسمه .env.example تضع فيه أسماء المتغيرات فقط بدون قيمها الحقيقية، لتسهيل الأمر على من يحمل المشروع بعدك. اطلب من الـ Agent فعل ذلك.
2. الصلاحيات: الـ AI Agent عادة يستطيع تنفيذ أوامر الـ Terminal، لكنه قد يحتاج منك تسجيل الدخول في GitHub عبر الـ CLI أولاً. إذا فشل في الرفع، استخدم الأمر gh auth login في التيرمينال الخاص بك.
3. المستودع (Repository): تأكد من إنشاء مستودع فارغ على GitHub أولاً لتزويد الـ Agent برابط الـ URL الخاص به (مثل https://github.com/username/repo-name.git) عندما يطلبه منك.

هل ترغب في تعديل أي جزء من البرومبت ليتناسب مع لغة برمجة معينة (مثل Python أو React)؟
